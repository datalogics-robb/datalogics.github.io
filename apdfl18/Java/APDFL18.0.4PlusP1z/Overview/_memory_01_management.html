Memory Management

PDFL objects can be associated with a large amount of unmanaged memory which exert little memory pressure on the runtime, which means there may be a long period before the garbage collector deletes these objects. The garbage collector is not aware of the size of the unmanaged or native objects in APDFL.

Thus, there may be a delay before garbage collection happens, and the large amount of memory that is associated with objects no longer in use is freed.

It is important then to release the unmanaged resources in APDFL as soon as a object is no longer in use.

To do this, call delete() method of the Object's class.
